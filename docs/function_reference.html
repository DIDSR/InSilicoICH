class insilicoICH.ground_truth_definition.phantoms.HeadPhantom(phantom_dir)¶
add_round_lesion(volume: int = 10, intensity: int = 50, material: str = 'white matter', eccentricity: float = 0.5, mass_effect: bool | float = 0.5, edema: bool | int = False, complexity: int = 3, overlap: float = 0.4, seed: int | None = None) → tuple¶

adds round lesion to img in random location within mask of size radius and intensity level intensity

See parameter descriptions below for further modifications that can be added:

Parameters:

  • volume – int or list of ints, volume of the sphere lesion in mL, if provided a list it will make concentric lesions

  • intensity – int or list of ints, intensity of the sphere lesion in HU, if provided a list it will make concentric lesions of intensities

  • material – which material region to insert lesion into, self.materials for options

  • eccentricity – between 0, 1 defines how elongated the lesions are, with 0 being spherical, 1 being very oblong

  • mass_effect – bool or float between [0, 1], if 0 or False no mass effect is applied, a mass effect > 0 but < 1 controls mass effect strength where 1 is a large degree of mass effect warping and 0.2 is a smaller amount of warping, see insert_with_mass_effect for more details

  • edema – bool or int, referring to the number of pixels thick of an edema layer to add around the lesion

  • complexity – int, number of ellipses to aid with random jiggle, 1 gives a single ellipsoid, increasing to 2 or 3 yields overlapping ellipsoids with a more complex shape.

  • overlap – float, allowed overlap with the white matter mask

  • seed – optional, defaults to None, set seed for reproducible lesion insertion

Returns:

img_w_lesion, lesion_vol, (z, x, y)

class insilicoICH.ground_truth_definition.phantoms.NIHPD_Head(phantom_dir, age: float, symmetric=False, csf_HU=10, gm_HU=40, wm_HU=30, skull_HU=900, shape=None)¶

loads MR brain atlas of mean age, downloaded from <https://www.bic.mni.mcgill.ca/~vfonov/nihpd/obj1> and saved in base_dir

Parameters:

  • base_dir – str, directory holding .nii files from https://www.bic.mni.mcgill.ca/~vfonov/nihpd/obj1

  • age – float, mean age of the atlas phantom to load. Note the brain atlases are stored as age ranges, thus the mean age is the mid point of the range (upper+lower)/2

  • symmetry – optional, the atlases are provided in their natural asymmetric or artificially generated symmetric state, default is asymmetric, see article for more details: Fonov V, Evans AC, Botteron K, Almli CR, McKinstry RC, Collins DL. Unbiased average age-appropriate atlases for pediatric studies. NeuroImage. 2011;54(1):313-327. doi:10.1016/j.neuroimage.2010.07.033

class insilicoICH.image_acquisition.Scanner(phantom: Phantom, studyname: str = 'default', studyid: int = 0, seriesname: str = 'default', seriesid=0, framework: str = 'CATSIM', output_dir: str | Path = None, materials: dict | None = None)¶

A class to hold CT simulation data and run simulations

Parameters:

  • phantom – Phantom class instance to be scanned, voxels in units of approximate CT Numbers [HU], typically in python coordinates (z, x, y) where z is perpendicular to the axial plane made by x and y. See <https://en.wikipedia.org/wiki/Hounsfield_scale> for some suggested values for common materials

  • studyname – str, study identifier to be saved in DICOM header

  • studyid – int, study identifier to be saved in DICOM header

  • seriesname – str, series identifier to be saved in DICOM header

  • seriesid – int, series identifier to be saved in DICOM header

  • output_dir – optional directory to save the intermediate and final simulation results, defaults to current working directory

  • framework – Optional, CT simulation framework options include [‘CATSIM’]

  • materials – Optional dictionary of {material name: HU value}, used for construction volume fraction maps in XCIST, see materials and thresholds from this XCIST example: https://github.com/xcist/phantoms-voxelized/blob/main/DICOM_to_voxelized/DICOM_to_voxelized_example_head.cfg

Returns:

None

run_scan(mA=200, kVp=120, startZ=None, endZ=None, views=None, table_speed=0, bhc=True)¶

Runs the CT simulation using the stored parameters.

Parameters:

  • mA – x-ray source milliamps, increases x-ray flux linearly, $noise ~ 1/sqrt(mA)$

  • kVp – x-ray source potential, increases x-ray flux nonlinearly and reduces contrast as increased

  • startZ – optional starting table position in mm of the scan, see self.start_positions

  • endZ – optional last position of scan in mm, see self.start_positions

  • views – number of angular views, for testing this can be reduced but will produced aliasing streaks

  • verbose – optional boolean, if True prints out status updates, if False they are suppressed.

  • table_speed – optional [float, str] str options include ‘Low’: 26.67, ‘Intermediate’: 48, ‘High’: 64, units in mm/s

  • bhc – optional [bool, str], if True applies polynomial beam hardening correction to correct for polychromatic cupping artifact, if default applied default XCIST bhc, caution this gives capping artifacts. Options include [True, False, ‘default’]
